Making your first (well, second) Repo!

A repository is the first step in getting code onto GiHub for collaborating with others. This will be a follow along activity, rather than a let you work on your own activity, but the directions have also been placed here for you to use as you see fit. 

In the upper-right corner of any page, select your profile picture, click “Your repositories then on the new page, click the green New button towards the top right.
Initialize with a readme, make sure it’s public, and name it “My First Code Repo!”
Once it’s created and you’re on the main page of the repo, click the period button on your keyboard. This will open the Web Editor. 
The web editor is VS Code, but doesn’t have the ability to “run” any files (there’s no terminal included, it’s only for text editing not for much else).
Inside the web editor, on the left vertical tabs, select the files icon. 
Select the new file button in the area next to your name (hover near name to show the icons)

Create a markdown file named after your username. (username.md)
Write your first and last name on line 1, and on line 2 put the name of the school you teach at. Line 3 should be empty (hit enter) and line 4 should be your favorite snack. 
Click on the Source control button in the left vertical tabs. 
Next to Changes, click the plus sign to `add` all changes to be staged and ready to commit.
Enter a message describing what you did. Imagine someone wants to know what changed in the code based on your commit message, but keep it under 7 words. Short, sweet, and informative. 
Click “Commit & Push”
